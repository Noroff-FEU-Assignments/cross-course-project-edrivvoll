Web Security

How would you advise a potential client to secure his/her website from web security attacks? Note: the word count for your answer should not be greater than 250 words

I would inform the potential client that there are many different security attacks through a website. Some of the most common ways is Cross-site scripting XSS, SQL injection and Cross-site request forgery CSRF.

  • Cross-site scripting XSS – This injects a harmful script through the browser. The script could send user login information to the attacker and the attacker could log in as that user.
  • SQL injection – Vulnerabilities in the SQL injection could make the attacker successfully run arbitrary SQL code on a database. With a successful attack the attacker could create new user with admin right and modify the server.
  • Cross-site request forgery CSRF – This is a method where the attacker uses the victim credentials without the victim knowing about it.
I would also inform the client about other threats such as: Clickjacking, Denial of service, Directory traversal, File inclusion and Command injection.
Login should be with a 2-factor login such as OneLogin or Microsoft Authenticator. With a password and verifying with a pre-setup verified authenticator app.
I would have advice the client to take precaution of all these threats and configure the server with https encryption. This will encrypt the data between the client and server, making it harder for attackers to get the information.
